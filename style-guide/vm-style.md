# VM & Instance — Response Style Guide

> DNA สกัดจาก Google Sheet FAQ กลุ่ม VM/Instance ของ OpenLandscape Cloud

## Structure

1. **เปิด:** "เรียน ผู้ใช้บริการ"
2. **รับทราบปัญหา** (ถ้ามี incident)
3. **อธิบายสาเหตุ** — สั้น 1-2 ประโยค
4. **แนะนำวิธีแก้** — เป็นขั้นตอน ชัดเจน
5. **แนบลิงก์:** "รายละเอียดตามลิงก์แนบ: [URL]"
6. **Escalate (ถ้าจำเป็น):** "หากยังพบปัญหา รบกวนแนบภาพ... ผ่าน Ticket"
7. **ปิด:** "ขอบคุณครับ"

## Tone
- ให้ข้อมูล, สุภาพ, เป็นทางการ
- "เบื้องต้นแนะนำให้..."
- "ทางทีมขอเรียนแจ้งว่า..."
- "ผู้ใช้บริการสามารถ..."
- ไม่ใช้: "รู้สึกว่า", "คิดว่า", "น่าจะ"

## Required Elements
- ต้องมีลิงก์ blog.openlandscape.cloud เมื่อมีคู่มือเกี่ยวข้อง
- Incident ต้องถาม instance name + public IP + ช่วงเวลา
- ถ้าเป็นข้อจำกัดของระบบ ต้องชี้แจงเหตุผล

## Variant: Ticket Reply Mode (ในระบบ internal.openlandscape.cloud)

เมื่อตอบใน Ticket (threaded) ให้ปรับรูปแบบดังนี้:

| สถานะ | รูปแบบ |
|---|---|
| **Reply แรก** (ไม่มีประวัติ) | ใช้ "เรียน ผู้ใช้บริการ" + ปิด "ขอบคุณครับ" ได้ |
| **Reply ถัดไป** (ใน thread เดียวกัน) | **ไม่ต้อง** ขึ้น "เรียน ผู้ใช้บริการ" และ **ไม่ต้อง** ปิด "ขอบคุณครับ" — ตอบตรงประเด็น |
| **Reply แจ้ง progress** | ปิดด้วย "ครับ/ค่ะ — หากมีความคืบหน้าจะแจ้งให้ทราบอีกครั้งครับ" |
| **Gender particle** | ใช้ "ครับ" หรือ "ค่ะ" ตามเพศเจ้าหน้าที่ผู้ตอบ |

> หมายเหตุ: ใน Ticket system จะมี Timestamp + Email sender กำกับอยู่แล้ว ไม่ต้องใส่ชื่อตัวเอง

## Key Phrases
- "เบื้องต้นรบกวนตรวจสอบ..."
- "แนะนำให้ทำ Snapshot ก่อนดำเนินการ"
- "หากยังพบอาการ รบกวนแนบภาพ..."
- "ทางทีมขอเรียนแจ้งว่า ระบบไม่สามารถ..."
