Step 1: Intake

ภาพรวมระบบ (Manual Overview)

Flow สำหรับรับสายลูกค้า Self Service โดยเริ่มจาก Intake, ระบุ Platform, ตรวจ Gate Domain/SSL ถ้าเกี่ยวข้อง, แยก Incident/Request, เปิด Ticket/Case, โทร Operation, Drop Line และปิดสายผ่าน Ticket/Email

Kory Gate Commercial Route-Out Email / Verify
กฎสำคัญ: Email ไม่ใช่ตัวตัดสินว่าลูกค้าอยู่ฝั่ง Kory หรือ Gate ต้องถาม Portal ก่อนเสมอ และถ้าเป็นเคส Instance / VM / Network ให้ใช้ Public IP ช่วยยืนยันฝั่งบริการ หาก Portal และ Public IP ขัดกัน ให้ส่งเข้า Email / Verify เพื่อขอข้อมูลเพิ่มและตรวจสอบก่อนเปิดเคส

Email-only last resort: ถ้ามีเพียง email และสุดวิสัยจริงๆ หลังพยายามขอ Portal/Public IP แล้ว ให้เปิดเคสไปที่ Kory ก่อน โดยระบุว่า platform ยังไม่ยืนยัน เพื่อให้ OP OLS ตรวจสอบต่อ

Main Intake Flow

1 Intake
Portal, Email, Public IP, อาการ/คำขอ
2 Route Platform
Kory, Gate, Commercial, Verify
3 Gate Pre-check
Domain/SSL ถ้าเกี่ยวข้อง
4 Incident / Request
แยกประเภทเคส
5 Open Case
internal หรือ pro-tracker
6 Call Operation
โทรแจ้งทุกเคสที่ลูกค้าโทรมา
7 Drop Line
แจ้งทีมทุกเคสที่ลูกค้าโทรมา
8 Call Closing
ติดตามผ่าน Ticket/Email

Standard หลังเปิด Ticket/Case

Incident

เปิด Ticket/Case, โทร Operation, Drop Line และแจ้งลูกค้าให้ติดตามผ่าน Ticket/Email ทุกครั้ง

Request

ถ้าลูกค้าโทรมา ให้เปิด Ticket/Case, โทร Operation และ Drop Line ทุกเคส เป็น standard เดียวกัน

Commercial / Alpha

ไม่เข้า Self Service flow ให้ Route-Out ไป noc@inet.co.th และปิดสายด้วย script เฉพาะ

Gate Domain / SSL Pre-check

ถ้าเป็นเคส Domain / SSL ให้ค้นหาใน internal ก่อน หากพบ Domain/SSL ให้เข้า flow ปกติ หากไม่พบ Domain/SSL แต่ Account ตรง ให้เปิดผ่าน Account และเข้า flow ปกติ หากไม่พบทั้ง Domain/SSL และ Account ไม่ตรง ให้เข้า Email / Verify ไม่เปิด Gate Ticket

Compact Diagram

Intake -> Platform? -> Kory -> Type -> Open Case -> Call Operation -> Drop Line -> Closing -> Gate -> Domain/SSL Check -> Type -> Open Ticket -> Call Operation -> Drop Line -> Closing -> Commercial / Alpha -> Route-Out noc@inet.co.th -> Closing -> Conflict / Email-only -> Email / Verify
