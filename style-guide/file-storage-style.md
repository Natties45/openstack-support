# File Storage — Response Style Guide

> DNA จาก File Storage FAQ — Bucket, Object Storage, Public Access, Swift, Enigma

## Structure
1. **เปิด:** "เรียน ผู้ใช้บริการ"
2. **ระบุคำถาม/ปัญหา**
3. **แนะนำขั้นตอน:** การตั้งค่า Bucket/Container, Public Access
4. **แนบลิงก์:** blog link
5. **Security:** เน้นความปลอดภัยของไฟล์สาธารณะ
6. **ปิด:** "ขอบคุณครับ"

## Tone
- เทคนิค, เน้นความปลอดภัย
- "ผู้ใช้บริการสามารถจัดการ File Storage ได้ผ่านหน้า Gate"
- "แนะนำให้ตรวจสอบการตั้งค่า Public Access ของ Bucket/ไฟล์"
- "เพื่อความปลอดภัย ควรปิดการเผยแพร่สาธารณะสำหรับส่วนที่ไม่จำเป็น"

## Required Elements
- File Storage ทั่วไป → https://blog.openlandscape.cloud/file-storage-feature/
- Public Access → แนะนำให้จำกัดเฉพาะไฟล์ที่จำเป็น
- Abuse/Malicious files → cross-ref ไป abuse.yaml + T&C
- Bucket/Container management → ชี้เมนูใน Gate

## Key Phrases
- "ผู้ใช้บริการสามารถอัปโหลดและจัดการไฟล์ผ่านบริการ File Storage บน Gate"
- "แนะนำให้ตรวจสอบการตั้งค่า Public Access และจำกัดเฉพาะไฟล์ที่ต้องการเผยแพร่"
- "หากพบไฟล์ที่เข้าข่ายไม่พึงประสงค์ รบกวนตรวจสอบและลบ/แก้ไขโดยเร็ว"
